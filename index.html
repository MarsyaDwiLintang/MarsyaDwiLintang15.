Pengertian Polimorfisme & Abstraksi

Polimorfisme adalah kemampuan objek untuk digunakan dalam berbagai bentuk, sedangkan abstraksi digunakan untuk menyederhanakan program dengan menyembunyikan detail yang tidak diperlukan. Contoh: Satu class Bentuk bisa digunakan oleh Persegi, Lingkaran, dll tanpa mengubah cara pemanggilannya.

Multiple Inheritance dengan Mixin

Dart tidak mendukung multiple inheritance secara langsung, tetapi menggunakan mixin untuk menambahkan banyak kemampuan dalam satu class. Contoh: Seekor bebek bisa terbang dan berenang karena menggunakan mixin.

01. Studi Kasus Mixin

Mixin digunakan untuk menambahkan kemampuan ke dalam class. Contoh: Burung bisa terbang, bebek bisa terbang dan berenang.

DART SOURCE CODE

//Nama : Marsya Dwi Lintang
//NIM  : 251410028
//Kelas: SI2A

mixin CanFly {
  void fly() {
    print('Can fly!');
  }
}

mixin CanSwim {
  void swim() {
    print('Can swim!');
  }
}

class Bird with CanFly {}
class Duck extends Bird with CanSwim {}

void main() {
  Bird bird = Bird();
  bird.fly();

  Duck duck = Duck();
  duck.fly();
  duck.swim();
}

02. Mixin dengan State

Mixin juga bisa menyimpan kondisi (state). Contoh: Objek bisa mengetahui apakah sudah terbang atau belum.

DART SOURCE CODE

//Nama : Marsya Dwi Lintang
//NIM  : 251410028
//Kelas: SI2A

mixin CanFly {
  bool _isFlying = false;

  void fly() {
    if (!_isFlying) {
      _isFlying = true;
      print('Started flying!');
    } else {
      print('Already flying!');
    }
  }
}

class Duck with CanFly {}

void main() {
  Duck duck = Duck();
  duck.fly();
  duck.fly();
}

03. Sealed Class

Sealed class membatasi turunan class. Contoh: Response API hanya bisa Success atau Error.

DART SOURCE CODE

// NIM   : 251410028

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

void main() {
  ApiResponse res = Success("Berhasil");
  print(res.runtimeType);
}

7. Getter & Setter dengan Validasi

Getter dan setter dapat digunakan untuk menambahkan validasi agar data yang dimasukkan sesuai aturan.

DART SOURCE CODE

class Produk {

  double _harga;

  Produk(this._nama, this._harga);

Praktik 2:Enkapsulasi Lengkap

Program ini membuat class User dengan field private (_username, _password, _email). Data diakses menggunakan getter dan setter. Setter memiliki validasi seperti username minimal 5 karakter, password minimal 8 karakter, dan email harus mengandung @, sehingga data lebih aman dan terkontrol.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028
// Kelas : SI2A

class User {
  String _username;
  String _password;
  String _email;
  
  User(this._username, this._password, this._email);

  String get username => _username;
  String get password => _password;
  String get email => _email;

  set username(String value) {
    if (value.length >= 5) {
      _username = value;
    } else {
      print("Username minimal 5 karakter");
    }

  }

  set password(String value) {
    if (value.length >= 8) {
      _password = value;
    } else {
      print("Password minimal 8 karakter");
    }

  }

  set email(String value) {
    if (value.contains("@")) {
      _email = value;
    } else {
      print("Email harus mengandung '@'");

    }

  }

}

void main() {
  var user1 = User("marsya123", "password123", "marsya@email.com");
  print("Username: ${user1.username}");
  print("Password: ${user1.password}");
  print("Email: ${user1.email}");
  print("\n--- Update Data ---");

  user1.username = "lintanggg";
  user1.password = "12345678";
  user1.email = "lintang@gmail.com";
  
  print("Username baru: ${user1.username}");
  print("Password baru: ${user1.password}");
  print("Email baru: ${user1.email}");

}

8. Encapsulation (Login System)

Encapsulation adalah konsep OOP yang membungkus data dan method dalam satu class serta membatasi akses langsung ke data tersebut.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028
// Kelas : SI2A

class User {
  String _username;
  String _password;

  User(this._username, this._password);

  String get username => _username;

  bool login(String username, String password) {
    return _username == username && _password == password;
  }

  void gantiPassword(String lama, String baru) {
    if (_password == lama) {
      if (baru.length >= 8) {
        _password = baru;
        print('Password berhasil diubah');
      } else {
        print('Password minimal 8 karakter');
      }
    } else {
      print('Password lama salah');
    }
  }
}

void main() {
  final user1 = User('admin_user', 'password0443');

  if (user1.login('admin_user', 'password0443')) {
    print('Login berhasil');
  }

  user1.gantiPassword('password0443', 'newPassword456');
}

Dari praktikum ini dapat disimpulkan bahwa konsep Object Oriented Programming (OOP) dalam Dart membantu membuat program lebih terstruktur dan aman. Penggunaan constructor memudahkan pembuatan objek dengan berbagai cara. Enkapsulasi digunakan untuk melindungi data dengan field private serta mengatur akses melalui getter dan setter. Selain itu, pembuatan class seperti RekeningBank menunjukkan bagaimana objek dapat digunakan untuk mensimulasikan proses nyata seperti transaksi deposit, penarikan, dan pengecekan saldo dengan validasi agar data tetap aman.

Bonus 3:Class Rekening Bank

Program ini membuat class RekeningBank dengan field private _nomorRekening dan _saldo. Class ini memiliki method deposit(), tarik(), dan cekSaldo() untuk melakukan transaksi. Program juga memiliki validasi agar saldo tidak menjadi negatif saat melakukan penarikan.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028
// Kelas : SI2A

class RekeningBank {

  String _nomorRekening;
  double _saldo;

  RekeningBank(this._nomorRekening, this._saldo);

  void cekSaldo() {
    print("Nomor Rekening: $_nomorRekening");
    print("Saldo saat ini: Rp $_saldo");
  }

  void deposit(double jumlah) {
    if (jumlah > 0) {
      _saldo += jumlah;
      print("Deposit berhasil: Rp $jumlah");
    } else {
      print("Jumlah deposit harus lebih dari 0");
    }
  }

  void tarik(double jumlah) {
    if (jumlah > 0 && jumlah <= _saldo) {
      _saldo -= jumlah;
      print("Penarikan berhasil: Rp $jumlah");
    } else {
      print("Saldo tidak cukup atau jumlah tidak valid");
    }
  }

}

void main() {

  RekeningBank rekening1 = RekeningBank("123456789", 1000000);

  rekening1.cekSaldo();
  rekening1.deposit(500000);
  rekening1.tarik(300000);
  rekening1.cekSaldo();

}
